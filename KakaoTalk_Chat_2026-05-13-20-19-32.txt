[1. 문제 정의]

기존 플래너 / To-Do List의 불편함

1) 화면 공간 차지
- Todo 앱을 켜두면 작업 화면을 가림
- 공부/작업과 병행 어려움
- 작은 사이즈 및 크기 조절 필요
- 투명도 기능 부족

2) 동기 유지 어려움
- “오늘 도파민 부족해서 하기 싫음”
- 체크아웃 후 방치
- Todo가 의무처럼 느껴짐
- 장기적으로 앱을 썩혀두게 됨

3) 감정 고려 부족
- 사용자의 현재 상태 고려 X
- 무기력할 때도 동일한 방식으로 목표 제시
- 심리적 부담 증가

4) 장기/단기 목표 혼합 문제
- 시험 공부 / 습관 / 성장 목표가 섞임
- 서로 성격이 다른 목표 관리 어려움

[2. 사용자 상황 정의]

누가 사용하는가?
- 생산성 추구형 사용자
- 계획형(J 성향)
- 각 잡고 사는 삶 선호
- 공부/자기관리 중심 사용자

언제 사용하는가?
- 시험기간
- 공부 시작 전
- 하루 계획 세울 때
- 집중 안 될 때
- 해야 하는 건 알지만 하기 싫을 때


[시나리오 A - 시험기간 학생]

상황:
- 여러 과목 공부 필요

문제:
- 시간 배분 어려움
- 우선순위 혼란
- 해야 할 일 많아 압박감 느낌

필요:
- 공부 시간 분배
- 현재 가장 중요한 할 일 표시
- 부담 없는 UI


[시나리오 B - 장기/단기 목표 사용자]

상황:
- 단기 할 일과 장기 목표를 따로 관리하고 싶음

문제:
- 장기 목표가 단기 할 일에 묻힘
- 서로 성격 다른 Todo 혼합됨

필요:
- 단기 Todo
- 장기 성장 Todo 분리


--------------------------------------------------

[3. Adaptiveness 정의]

1) 감정 상태 기반 변화

사용자 상태:
- 😵 무기력
- 😐 보통
- 🔥 집중됨

상태에 따라:
- 메시지 변화
- Todo 강도 변화
- 추천 행동 변화

예시:
- “오늘은 5분만 해도 충분해.”
- “완벽보다 지속이 중요하다.”


2) 동기 유지 방식 변화

할 일을 오래 안 하면:
- 캐릭터가 썩어감
- 공간이 어두워짐
- 식물이 시들음

(듀오링고 느낌)

반대로 완료 시:
- 성장
- 회복
- 칭찬 효과


3) 화면 모드 변화

집중 상태:
- 미니모드 자동 전환
- 반투명
- 현재 할 일만 표시

계획 상태:
- 전체 Todo 표시


--------------------------------------------------

[4. 기능 브레인스토밍]

기능 후보

1) 미니 플로팅 모드
- 작은 위젯 형태
- 드래그 가능
- 투명도 조절

2) 감정 선택 시스템
- 앱 시작 시 오늘 상태 선택

3) 장기 / 단기 Todo 분리
- Today
- Long Term

4) 동기 유지 캐릭터
- 할 일 안 하면 상태 악화
- 완료하면 성장

5) 격언 시스템
- 상황별 메시지 출력

예시:
- “동기보다 중요한 건 시스템이다.”
- “지금의 10분이 미래를 만든다.”

6) 시험기간 집중 모드
- 과목별 시간 분배
- 집중 타이머
- 오늘 공부량 표시


--------------------------------------------------

MVP 핵심 기능

4) 감정 상태 선택
